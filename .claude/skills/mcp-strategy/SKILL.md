# MCP Strategy & Usage Guidelines

各MCPサーバーは以下の目的で戦略的に使い分けてください。推測でコードを書かず、必ずこれらのツールで裏付けを取ること。

| MCP Server                 | Target               | Role & Reason                                                                                                                                               |
| :------------------------- | :------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **context7**         | Next.js, Better-auth | **【知識の正当化】**`<br>`AIの持つ古い知識を上書きします。Next.js公式やライブラリのドキュメントを直接読み込み、幻覚（ハルシネーション）を防ぎます。 |
| **MySQL**            | TiDB, Drizzle        | **【事実確認】**`<br>`TiDBに接続し、マイグレーション結果やテストデータの整合性を「推測」ではなく「事実」として確認させます。                        |
| **Puppeteer**        | Playwright, Tailwind | **【視覚確保】**`<br>`Playwrightのテストを書く際、実際のDOM構造（class名や階層）を見て、壊れにくいセレクタを選定させます。                          |
| **Next.js DevTools** | App Router           | **【状態把握】**`<br>`サーバーコンポーネントのエラーやルーティングの状態を監視し、デバッグを加速させます。                                          |
| **Brave Search**     | General              | **【トラブルシューティング】**`<br>`公式ドキュメントに載っていないエラー回避策やGitHub Issueを探すために使用します。                                |

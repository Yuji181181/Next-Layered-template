# Review Judgment Criteria

AIレビュワー（CodeRabbit, GitHub Copilot）からの指摘を評価する際の基準です。
あなたは**裁判官**として振る舞い、すべての指摘に従うのではなく、以下の基準で採用・却下を判断してください。

## ✅ ACCEPT (修正すべき指摘)

* **Security**: SQLインジェクション、XSS、認証不備などのセキュリティリスク。
* **Bug**: 実行時エラー、境界値の処理漏れ、ロジックの明白な誤り。
* **Performance**: N+1問題、不要な再レンダリングなどの明確なパフォーマンス低下。
* **Type Safety**: TypeScriptの型安全性に関わる指摘。

## ❌ REJECT (却下すべき指摘)

* **Architecture Violation**: レイヤードアーキテクチャの責務（例：API層にビジネスロジックを書けという指示）に反する提案。
* **Style Preference**: Biomeの設定に反する、あるいは単なる好みのスタイル指摘。
* **Out of Scope**: 今回の変更に関係のない箇所へのリファクタリング要求。
* **Validation Error**: レビュワーが仕様(`docs/SPEC.md`)を誤解している場合。

**※ 却下する場合のアクション:**
コードは修正せず、思考プロセスログ（またはPRコメント）に「〜の理由により、この指摘は適用しません」と明確に残すこと。
